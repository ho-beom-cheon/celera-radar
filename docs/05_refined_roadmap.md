# 05. 셀러레이더 세분화 로드맵 v2

- 문서 버전: v0.2
- 기준일: 2026-07-03
- 현재 상태: Docker, PostgreSQL, Spring Boot, React 기본 기동 완료
- 목적: 현재 골조 이후 기능 구현 순서를 더 작게 분해하고, 스마트스토어 API를 앱인토스보다 먼저 진행하는 변경 로드맵을 확정한다.

---

## 1. 확정된 제품 방향

셀러레이더는 초보 스마트스토어/위탁판매 셀러가 상품 후보를 검토하기 위해 사용하는 웹 SaaS다.

서비스는 다음을 자동화한다.

```text
키워드 수집
→ 네이버 쇼핑 가격 스냅샷
→ 트렌드 스냅샷
→ 도매 상품 CSV 매칭
→ 예상 마진 계산
→ 상품 검토 점수 산출
→ 알림
→ 스마트스토어 상품/주문/정산 연동
```

단, 다음은 자동화하지 않는다.

```text
상품 자동 등록
판매가 자동 변경
광고 자동 집행
상세페이지 자동 게시
판매 성공 보장
```

---

## 2. 변경된 큰 순서

기존에는 앱인토스 Lite가 스마트스토어 API보다 앞에 있었다. v2에서는 순서를 바꾼다.

```text
이전:
웹 SaaS MVP → 앱인토스 Lite → 스마트스토어 API

변경:
웹 SaaS MVP → 스마트스토어 API → 앱인토스 Lite
```

변경 이유:

1. 앱인토스는 유입 채널이고, 스마트스토어 API는 제품 본체의 가치를 높이는 기능이다.
2. 스마트스토어 API 연동 후 앱인토스를 만들면 토스 안에서 보여줄 요약 가치가 더 강해진다.
3. 단순 마진 계산기보다 “내 상품 마진 위험 요약”이 결제 설득력이 높다.

---

> DB 상세 설계는 `docs/10_database_design.md`를 기준으로 한다. P1~P3의 migration은 해당 문서의 DDL 초안을 우선 적용한다.

```text
item 단위 상품 정보 저장
```

### DB

```text

```


```text

이미지 파일 저장 없이 image_url만 사용
```

## P3. 가격/경쟁 분석

### 목표

키워드별 가격대와 1차 경쟁강도를 계산한다.

### 계산 예시

```text
min_price = 상위 상품 최저가
max_price = 상위 상품 최고가
avg_price = 상위 상품 평균가
result_count = 검색 결과 수
competition_level = LOW / MEDIUM / HIGH
```

### 1차 경쟁강도 기준

```text
검색 결과 수 10,000개 이상: HIGH
검색 결과 수 3,000~9,999개: MEDIUM
검색 결과 수 3,000개 미만: LOW
```

### 완료 기준

```text
키워드 목록에 가격대 표시
키워드 상세에 경쟁강도 표시
계산 로직 단위 테스트 존재
```

## P6. 마진/상품 검토 점수

### 목표

네이버 쇼핑 가격대와 도매 공급가를 비교해 예상 마진과 상품 검토 점수를 계산한다.

### 산식 예시

```text
예상 판매가 = 네이버 상위 상품 평균가 또는 사용자가 선택한 기준가
총 원가 = 공급가 + 배송비 + 포장비 + 기타비용 + 예상 수수료
예상 순이익 = 예상 판매가 - 총 원가
예상 마진율 = 예상 순이익 / 예상 판매가 * 100
```

### 상품 검토 점수

```text
상품 검토 점수 =
트렌드 점수
+ 가격대 점수
+ 마진 점수
+ 경쟁강도 점수
- 위험 카테고리 감점
- CS/배송 리스크 감점
```

### 완료 기준

```text
추천 대신 검토 후보로 표시
점수 근거가 화면에 표시
80점 이상 추천 검토 후보
65~79점 검토 후보
64점 이하 보류
위험 카테고리는 추천 제외
```
