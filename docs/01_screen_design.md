# 셀러레이더 화면설계서

- 문서 버전: v0.1
- 작성일: 2026-07-02
- 대상 단계: Web SaaS v0.1~v0.4, 앱인토스 Lite v1.0
- 서비스 가칭: 셀러레이더
- 문서 목적: Codex Pro 구현 전 화면 범위, 사용자 흐름, 입력값, 상태, 검증 기준을 고정한다.

---

## 1. 서비스 화면 전략

셀러레이더는 하나의 백엔드와 하나의 DB를 공유하되, 사용 화면을 두 가지로 나눈다.

| 채널 | 역할 | 핵심 기능 | 초기 우선순위 |
|---|---|---|---|
| Web SaaS | 본체, 분석/관리용 | 키워드 분석, CSV 업로드, 추천점수, 알림, 리포트 | 1순위 |
| 앱인토스 Lite | 유입/검증용 | 빠른 마진 계산, 오늘 추천 카드, 관심 키워드 일부 | 2순위 |
| 별도 모바일 앱 | 향후 확장 | 푸시 알림, 간단 조회 | 보류 |

화면 설계 원칙은 다음과 같다.

1. 사용자는 “상품 추천”보다 “검토 후보”를 받는다고 인식해야 한다.
2. 모든 추천 결과에는 추천 이유와 주의점을 같이 보여준다.
3. 네이버 API는 화면 진입 시 실시간 호출하지 않고, 서버 배치/캐시 결과를 보여준다.
4. Web SaaS는 표와 필터 중심, 앱인토스는 카드와 계산기 중심으로 설계한다.
5. MVP에서는 AI 기능을 넣지 않는다.

## 3. 주요 사용자 흐름

### 3.1 Web SaaS - 키워드 분석 흐름

```text
로그인
→ 키워드 레이더
→ 키워드 등록
→ 다음 배치 대기
→ 분석 완료
→ 가격대/경쟁강도/트렌드 확인
→ 관심 또는 제외 처리
```

### 3.2 Web SaaS - 도매 CSV 기반 상품 후보 발굴

```text
도매 CSV 업로드
→ 컬럼 매핑
→ 파싱 검증
→ 네이버 가격대 캐시와 매칭
→ 예상 마진 계산
→ 추천점수 산출
→ 후보 목록 확인
→ 알림 조건 저장
```

### 3.3 앱인토스 Lite - 가벼운 검증 흐름

```text
토스 앱 진입
→ 셀러레이더 Lite 실행
→ 오늘 추천 후보 확인
→ 빠른 마진 계산
→ 결과 저장
→ 관심 키워드 등록
→ 유료 기능 안내
```

## 5. 공통 UI 규칙

### 5.1 상태 배지

| 상태 | 의미 | 표시 |
|---|---|---|
| RECOMMENDED | 추천 후보 | 추천 |
| REVIEW | 검토 후보 | 검토 |
| HOLD | 보류 | 보류 |
| EXCLUDED | 제외 | 제외 |
| PENDING | 분석 대기 | 대기 |
| ANALYZED | 분석 완료 | 완료 |
| FAILED | 분석 실패 | 실패 |

### 5.2 추천 표현 원칙

사용 가능한 문구:

```text
데이터 기반 검토 후보입니다.
트렌드/가격/마진 기준으로 검토할 만합니다.
실제 판매량 데이터가 아니므로 최종 판단은 직접 확인이 필요합니다.
```

사용 금지 문구:

```text
무조건 팔립니다.
돈 되는 상품입니다.
월 수익이 보장됩니다.
이 상품을 올리면 수익이 납니다.
```

### 5.3 기본 빈 상태

| 상황 | 메시지 |
|---|---|
| 키워드 없음 | 아직 등록한 키워드가 없습니다. 관심 상품 키워드를 추가하세요. |
| 분석 대기 | 등록한 키워드는 다음 배치에서 분석됩니다. |
| CSV 없음 | 도매 상품 CSV를 업로드하면 예상 마진을 계산할 수 있습니다. |
| 추천 없음 | 현재 조건에 맞는 추천 후보가 없습니다. 조건을 완화하거나 키워드를 추가하세요. |
| API 오류 | 외부 데이터 수집에 실패했습니다. 마지막 성공 데이터를 표시합니다. |

## 6. 상세 화면 설계

## WEB-001 로그인/회원가입

### 목적

Web SaaS 사용자의 계정을 생성하고 JWT 인증 세션을 발급한다.

### 주요 컴포넌트

| 영역 | 요소 | 설명 |
|---|---|---|
| 헤더 | 서비스명 | 셀러레이더 |
| 입력 | 이메일 | 필수, 이메일 형식 |
| 입력 | 비밀번호 | 필수, 8자 이상 |
| 체크 | 이용약관 동의 | 회원가입 시 필수 |
| 버튼 | 로그인 | 기존 사용자 |
| 버튼 | 회원가입 | 신규 사용자 |

### 검증

- 이메일 형식이 아니면 저장하지 않는다.
- 비밀번호는 8자 이상으로 제한한다.
- 회원가입 성공 후 대시보드로 이동한다.

## WEB-002 대시보드

### 목적

사용자가 오늘 확인해야 할 분석 결과를 한 화면에 요약한다.

### 와이어프레임

```text
┌──────────────────────────────────────────────┐
│ 셀러레이더                         [설정]   │
├──────────────────────────────────────────────┤
│ 오늘의 분석 요약                             │
│ 추천 12개 | 검토 34개 | 보류 86개 | 실패 2개 │
├──────────────────────────────────────────────┤
│ 오늘의 추천 후보 TOP 5                       │
│ [상품/키워드] [점수] [예상마진] [주의]       │
├──────────────────────────────────────────────┤
│ 키워드 분석 현황                             │
│ 전체 430 | 완료 410 | 대기 18 | 실패 2       │
├──────────────────────────────────────────────┤
│ 최근 알림                                    │
│ - 차량용 틈새 수납함: 점수 82                │
└──────────────────────────────────────────────┘
```

### 데이터

| 항목 | 출처 |
|---|---|
| 추천/검토/보류 수 | candidate_summary API |
| TOP 후보 | candidate API |
| 키워드 분석 현황 | keyword analysis API |
| 최근 알림 | alert API |

### 액션

- 후보 클릭: WEB-006 이동
- 키워드 현황 클릭: WEB-003 이동
- 알림 클릭: WEB-010 이동

## WEB-003 키워드 목록

### 목적

관심 키워드를 등록하고 분석 상태를 확인한다.

### 주요 컬럼

| 컬럼 | 설명 |
|---|---|
| 키워드 | 예: 차량용 수납함 |
| 카테고리 | 차량용품/데스크/자취 등 |
| 우선순위 | HIGH/MEDIUM/LOW |
| 분석 상태 | 대기/완료/실패 |
| 마지막 분석일 | yyyy-mm-dd hh:mm |
| 트렌드 점수 | 0~30 |
| 경쟁 점수 | 0~25 |
| 액션 | 상세/수정/삭제 |

### 입력 폼

| 필드 | 필수 | 규칙 |
|---|---|---|
| keyword | Y | 2~50자 |
| category | Y | 사전 정의 카테고리 |
| priority | N | 기본 MEDIUM |

### 무료/유료 제한

| 플랜 | 키워드 등록 가능 수 |
|---|---:|
| FREE | 3개 |
| BASIC | 30개 |
| PRO | 100개 |

## WEB-004 키워드 상세

### 목적

키워드별 쇼핑 가격대, 트렌드, 경쟁강도, 추천 판단을 보여준다.

### 화면 구성

```text
[키워드] 차량용 먼지 브러쉬
상태: 분석 완료 | 마지막 분석: 2026-07-02 07:12

1. 요약 카드
- 최저가: 4,900원
- 평균가: 12,300원
- 검색 결과 수: 18,230
- 경쟁강도: 보통
- 트렌드: 상승

2. 트렌드 차트
최근 30일 ratio 표시

3. 상위 상품 테이블
상품명 | 최저가 | 쇼핑몰 | 브랜드 | 카테고리

4. 판단
검토 후보: 가격대는 형성되어 있으나 상위 리뷰 경쟁 확인 필요
```

### 주의 문구

키워드 상세에는 항상 다음 문구를 노출한다.

```text
데이터랩 수치는 검색 클릭 추이 기반이며 실제 판매량을 의미하지 않습니다.
```

### 목적

| 섹션 | 내용 |

## WEB-007 도매 CSV 업로드

### 목적

사용자가 확보한 도매 상품 파일을 업로드해 상품 후보를 생성한다.

### 지원 파일

| 항목 | 기준 |
|---|---|
| 확장자 | .csv, .xlsx는 v0.3 이후 선택 |
| 인코딩 | UTF-8 우선, CP949 자동 감지 시도 |
| 최대 행 수 | FREE 100행, BASIC 3,000행, PRO 20,000행 |
| 필수 컬럼 | 상품명, 공급가 |
| 선택 컬럼 | 배송비, 카테고리, 이미지URL, 상품URL |

### 컬럼 매핑 화면

| 내부 필드 | CSV 컬럼 예시 |
|---|---|
| product_name | 상품명, 제품명, title |
| supply_price | 공급가, 도매가, cost |
| shipping_fee | 배송비, delivery_fee |
| category | 카테고리, 분류 |
| product_url | 상품URL, link |

## WEB-008 CSV 매칭 결과

### 목적

CSV 파싱 결과, 오류 행, 예상 마진 계산 결과를 보여준다.

### 상태

| 상태 | 의미 |
|---|---|
| PARSED | 파싱 완료 |
| INVALID | 필수값 누락 |
| MATCHED | 네이버 가격대 매칭 완료 |
| LOW_MARGIN | 목표 마진 미달 |
| RISK_EXCLUDED | 위험 카테고리 제외 |

### 오류 예시

```text
12행: 공급가가 숫자가 아닙니다.
37행: 상품명이 비어 있습니다.
84행: 배송비가 음수입니다.
```

## WEB-009 마진 계산기

### 목적

판매가, 공급가, 수수료, 배송비, 광고비, 쿠폰을 입력해 예상 순이익과 권장 판매가를 계산한다.

### 입력 필드

| 필드 | 필수 | 기본값 |
|---|---|---|
| sale_price | Y | 없음 |
| supply_price | Y | 없음 |
| platform_fee_rate | Y | 4% |
| shipping_fee | N | 0 |
| packaging_fee | N | 0 |
| ad_cost_per_order | N | 0 |
| coupon_discount | N | 0 |
| target_margin_rate | Y | 25% |

### 계산식

```text
platform_fee = sale_price * platform_fee_rate
expected_cost = supply_price + platform_fee + shipping_fee + packaging_fee + ad_cost_per_order + coupon_discount
profit = sale_price - expected_cost
margin_rate = profit / sale_price * 100
recommended_price = expected_cost_without_sale_price_dependent_fee / (1 - target_margin_rate - platform_fee_rate)
```

### 결과 표시

| 결과 | 설명 |
|---|---|
| 예상 순이익 | 원 단위 |
| 예상 마진율 | % |
| 상태 | 안전/주의/위험 |
| 권장 판매가 | 목표 마진율 기준 |
| 허용 광고비 | 목표 마진 유지 가능 금액 |
| 허용 쿠폰 | 목표 마진 유지 가능 금액 |

## 9. 화면 완료 기준 Definition of Done

1. 각 화면은 로딩/빈 상태/오류 상태를 가진다.
2. 외부 API 호출 실패 시 마지막 캐시 데이터를 보여준다.
3. 추천 결과에는 반드시 추천 이유와 주의점을 표시한다.
4. 무료/유료 제한이 화면에서 명확하다.
5. 앱인토스 화면은 외부 이동 없이 핵심 기능을 완료한다.
6. 모바일 폭 360px에서 주요 기능이 깨지지 않는다.
7. 접근성 기준으로 버튼 라벨, 폼 라벨, 키보드 이동이 가능해야 한다.
