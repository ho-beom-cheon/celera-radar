# 09. 의사결정 로그 v2

- 문서 버전: v0.2
- 기준일: 2026-07-03
- 목적: 프로젝트 진행 중 주요 의사결정을 기록하여 Codex 작업과 추후 설계 변경의 기준점으로 사용한다.

---

## D-001. 서비스 포지션

결정:

```text
셀러레이더는 “판매 성공 보장 서비스”가 아니라 “데이터 기반 상품 검토 후보 서비스”다.
```

이유:

```text
외부 데이터는 실제 판매량/수익을 보장하지 않음
정책/법적 리스크 감소
사용자 기대치 관리
```

영향:

```text
UI 문구에서 추천, 수익, 성공 보장 표현을 피한다.
점수명은 상품 검토 점수로 한다.
```

---

## D-002. 서비스 구조

결정:

```text
웹 SaaS가 본체이고, 앱인토스 Lite는 나중 유입/요약 채널이다.
```

이유:

```text
CSV 업로드, 대량 분석, 리포트, 스마트스토어 연동은 PC 웹에 적합
앱인토스는 가벼운 진입점으로 적합
```

---

## D-003. 앱인토스와 스마트스토어 API 순서 변경

결정:

```text
스마트스토어 API 연동을 앱인토스 Lite보다 먼저 진행한다.
```

이유:

```text
본체 기능 가치를 먼저 강화
앱인토스 출시 시 보여줄 요약 데이터가 강해짐
단순 계산기보다 내 상품 마진 위험 요약이 더 설득력 있음
```

---

## D-004. 초기 도매 데이터 방식

결정:

```text
초기에는 도매 API가 아니라 CSV/XLSX 업로드를 사용한다.
```

이유:

```text
API 승인/정책/호출 제한 리스크 회피
Codex 구현 난이도 낮음
사용자별 다양한 공급사 파일 대응 가능
```

---

## D-005. 외부 API 호출 구조

결정:

```text
사용자 요청마다 실시간 호출하지 않고, snapshot/cache 구조를 사용한다.
```

이유:

```text
API 호출 한도 보호
응답속도 안정화
상용화 가능성 확보
```

---

## D-006. 상품 이미지 처리

결정:

```text
상품 이미지는 다운로드하지 않고 image_url만 저장한다.
```

이유:

```text
저장소 비용/저작권/재가공 리스크 감소
분석 도구 목적에 충분
```

---

## D-007. AI 사용

결정:

```text
MVP에서는 AI를 사용하지 않는다.
```

이유:

```text
토큰 비용 최소화
추천 로직의 재현성 확보
Codex 구현 범위 축소
```

향후 사용 가능 영역:

```text
추천 사유 문장 생성
상세페이지 초안
월간 리포트 요약
```

## D-008. 디자인 전략

결정:

```text
핵심 기능 구현 후 대규모 디자인 업데이트를 진행한다.
```

이유:

```text
기능 흐름이 아직 바뀔 수 있음
초기부터 디자인에 비용을 많이 쓰면 재작업 증가
Codex 작업 효율 저하 방지
```

단, 현재부터 지킬 원칙:

```text
공통 컴포넌트 사용
하드코딩 스타일 최소화
화면 로직과 표현 컴포넌트 분리
```
